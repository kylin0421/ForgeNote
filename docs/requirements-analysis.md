# 需求分析说明

## 赛题理解

赛题要求构建“基于大模型的个性化资源生成与学习多智能体系统”。核心不是单点问答，而是让系统根据学生差异持续理解学习状态，主动生成和推送合适的课程资源，并通过多智能体协作降低内容幻觉和资源失配风险。

智学工坊选择在成熟 AI notebook 底座上开发，原因是高校学习天然依赖资料、笔记、引用和知识库。原 notebook 能很好地承载资料，但偏通用、偏被动；本项目的改造重点是把资料库升级为主动学习系统。

## 用户与场景

目标用户是高校学生，尤其是人工智能、计算机、电子信息等课程学习者。

典型场景：

- 学生上传课程讲义、论文、教材章节、实验说明或网页资料。
- 学生通过自然语言说明专业背景、学习目标、薄弱点和资源偏好。
- 系统分析资料与画像，生成个性化讲解、练习、导图、阅读材料和代码实验。
- 学生在学习过程中提问、完成测验、查看资源，系统据此更新画像和学习路径。

## 痛点分析

| 痛点 | 影响 | 系统应对 |
| --- | --- | --- |
| 资料繁杂且质量不一 | 学生难以判断先看什么、怎么看 | 资源搜索智能体按学习目标搜集候选资料，并由学生选择接受来源 |
| 学生基础差异大 | 统一讲解难以匹配个人节奏 | 学习画像记录知识基础、认知风格和薄弱点 |
| 学习资料缺少结构化加工 | 课程内容难以转化为可执行任务 | 生成讲解、导图、测验、闪卡、代码实验等多类型资产 |
| 大模型回答可能幻觉 | 学术学习要求来源可靠 | 来源约束、引用、语义索引和安全智能体共同降低风险 |
| 长任务等待不透明 | 多模态生成耗时，演示体验差 | 任务队列、状态浮窗、SSE 编排事件和日志查看 |

## 功能需求

### 1. 对话式学习画像

系统需要摒弃长表单，以自然语言输入和学习行为构建画像。画像维度包括：

- 专业背景
- 学习目标
- 知识基础
- 认知风格
- 易错点/薄弱点
- 资源偏好
- 学习进度
- 练习反馈

画像保存在学习记录内的“学习画像”来源中，可被 RAG、资源搜索和资产生成复用。

### 2. 多智能体资源生成

系统需要明确体现多智能体协作。智学工坊拆分为：

- 画像智能体
- 课程智能体
- 资源搜索智能体
- 资产生成智能体
- 练习智能体
- 路径规划智能体
- 辅导智能体
- 评估智能体
- 安全智能体

资源类型至少覆盖：

- 课程学习讲解
- 测验题
- 闪卡
- 思维导图
- 拓展阅读
- 代码实验
- 播客/音频讲解

### 3. 个性化学习路径与推送

系统需要根据画像、学习进度、已接受来源和练习表现，给出下一步学习顺序。路径不是固定课程目录，而是可随画像和反馈调整。

### 4. 智能辅导

系统需要支持即时答疑。智学工坊基于 notebook 内的来源与引用能力，提供来源感知问答，并在学习页展示上下文引用与 token 预算。

### 5. 学习效果评估

系统需要通过学习事件、练习结果、资源使用反馈持续更新画像和推荐策略。当前实现支持学习事件记录和画像更新，为后续更细粒度学习分析预留接口。

## 非功能需求

| 非功能要求 | 实现方式 |
| --- | --- |
| 界面美观、交互清晰 | Next.js + Radix UI + Tailwind 风格组件，左中右三栏学习工作台和 Studio 资产入口 |
| Markdown 与多模态展示 | 讲解、测验、导图、代码实验等以结构化文本和卡片预览展示 |
| 防幻觉与安全过滤 | 来源约束生成、安全智能体、引用、语义索引、内容质量检查 |
| 响应时间合理 | 长任务后台化，任务浮窗显示状态；流式接口可展示 agent stage |
| 开源合规 | README 和开源说明文档中标注 notebook 底座、许可证和主要依赖 |

## 成功标准

- 能完整运行前端、后端和数据库。
- 能创建学习记录、添加课程资料、维护学习画像。
- 能展示多智能体生成资源的入口、流程和产物。
- 能生成至少 5 类个性化资源。
- 能清楚说明防幻觉、安全、进度追踪和开源合规设计。
- 有可复现测试结果和演示流程。

## 补充：学习状态可视化

为避免学习画像只停留在文本摘要，系统需要提供独立的学习状态可视化入口。当前实现中，学习记录页提供“学习曲线”面板，用于展示最近 14 天学习量、学习质量、测验正确率、每日明细和近期学习建议。该能力承接“学习效果评估”需求，把学习行为、练习结果和资源使用反馈转化为用户能直接理解的趋势和下一步行动。
